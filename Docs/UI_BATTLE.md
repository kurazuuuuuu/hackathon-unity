# 戦闘画面要件
* キャンバスサイズは4K(3840x2160)
* スマートフォンでもプレイする前提の画面設計
* UIの文字は可読性を優先し、アウトラインを積極的に使用する

* プレイヤー1（自分）：手前側
* プレイヤー2（相手）：奥側

* 2Dシーンであるが3Dオブジェクトも使用して奥行きを再現する

* プレイヤー1の手札は画面下部に表示
    * 山札は画面右側
    * 名前・体力が画面左側
    * 体力は数字とバー両方で表示する
* プレイヤー2の手札は内容を表示せず、カードの枚数のみ表示する
    * 山札は画面左側
    * 名前・体力が画面奥側
    * 体力は数字とバー両方で表示する

* 画面左側にバトルのログを表示する
* 画面中央にターン数（”ターン1”など）を表示する
* 画面右側にターンスキップボタンを配置する

* **原神の七星召喚をモチーフとしたデザインを行い、必要に応じて画像生成も可**

* カード本体は `Assets/Resources/Prefabs/Cards` にあるものを参照し、比率やサイズを計算する。小さすぎてはいけない。

## アニメーション
* 攻撃・被弾・特殊効果発動・特殊効果被弾時にアニメーションやVFXを再生するようにする
* アニメーションはエージェント単体でも実装可能なため行う

### アニメーション詳細
#### 攻撃
* 攻撃対象カードへ直接攻撃するようなアニメーション
* リニアではなく、加速度を感じるスピードで攻撃する
* 攻撃対象カードに到達したら、カードを大きく膨らませるようなアニメーション

#### 被弾
* 攻撃対象カードに到達したら、カードを大きく膨らませるようなアニメーション

#### 特殊効果発動
* 発動したカードを発光させ、カードを大きく膨らませるようなアニメーション

#### 特殊効果被弾
* 被弾したカードを大きく膨らませるようなアニメーション
