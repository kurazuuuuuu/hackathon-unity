# フォントレンダリング黙示録 (Font Rendering Apocalypse)

## 概要
本プロジェクトにおける「DotGothic16」フォントのレンダリング問題と、その解決に至るまでの苦闘の記録。
SDF (Signed Distance Field) とドット絵フォントの相性の悪さが引き起こした数々の現象について記す。

## 第1章：箱の出現 (The Boxes)
**現象**: 文字の周りに四角い枠（ボックス）がうっすらと表示される。
**原因**:
*   **Texture Wrap Mode**: `Repeat` になっていたため、アトラス内の隣接する文字や端のピクセルを拾ってしまっていた。
*   **Padding不足**: SDF生成時の余白が足りず、隣の文字のデータが干渉した。

## 第2章：塵への回帰 (Return to Dust)
**現象**: 文字が崩壊し、点々（塵）のようなノイズしか表示されなくなる。
**原因**:
*   **Gradient Scaleの暴走**: 文字をくっきりさせようと `Gradient Scale` を `50` などの高値に設定していた。
*   **Sampling Point Sizeの縮小**: その後、設定を `Auto` にしたことでサンプリングサイズが小さくなった。
*   **結果**: 小さなSDFデータに対して過剰な閾値判定（Scale 50）が行われ、有効なピクセルがほぼ全て消滅した。

## 第3章：虚無 (The Void)
**現象**: 「ターン情報」などの一部の文字が完全に消える、またはテクスチャが見切れる。
**原因**:
*   **Atlas Overflow**: 日本語の全文字セット（7000字以上）に対し、`Sampling Point Size: 64` という巨大なサイズを指定した。
*   **結果**: 4096 x 4096 の巨大テクスチャをもってしても文字が入りきらず、生成が途中で打ち切られた。

## 第4章：穴あき (The Holes)
**現象**: 文字の中に虫食いのような穴が空く。
**原因**:
*   **Filter Modeの不一致**: ドット絵だからとテクスチャを `Point` フィルタにした。
*   **SDFの仕様**: SDFは「グラデーション」を補間して輪郭を作る技術であるため、`Bilinear` フィルタが必須。`Point` にするとグラデーションが階段状になり、計算が破綻した。

## 最終章：救済 (Salvation - Raster Mode)
**結論**:
ドット絵フォント（特に日本語のような複雑なもの）において、SDFは調整難易度が高すぎる。
「ドットをドットとしてそのまま出す」のが正義である。

**解決策**: **Raster (Bitmap) モード**
*   **Render Mode**: `RASTER`
*   **Filter Mode**: `Point`
*   **Shader**: `TextMeshPro/Mobile/Bitmap`

これにより、SDFの呪縛から解き放たれ、完全なドット絵が表示されるようになった。
